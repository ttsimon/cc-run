# 这是什么

**ccr** 把选定 provider 的环境变量注入**当前终端**再拉起 `claude`，从而可同时多开、各用不同后端，互不干扰，且**不改全局配置**。

## 它解决什么问题

如果你经常在多个 AI 后端之间切换——比如用 A 写代码、用 B 做审查、用 C 做规划——每次改环境变量或改全局配置很繁琐，也很容易搞混。

ccr 的方案很简单：**一个终端 tab = 一个后端**。开三个 tab 各跑一次 `ccr <name>`，三个后端并行工作，谁也不用等谁，也不会互相覆盖。

## cc-switch vs ccr

| 维度 | cc-switch | ccr |
|------|-----------|-----|
| 切换粒度 | 全局 | 当前终端 |
| 同时生效数 | 1 个 | 不限（多 tab） |
| 改全局配置吗 | 是 | 否 |
| 多开方式 | 不支持 | 开多个 tab 各跑一次 |

**一句话**：cc-switch 是"这台机器用这个后端"，ccr 是"这个 tab 用这个后端"。两者各管各的，不冲突。

## 什么时候用

- 你装了 cc-switch 管理多个 provider，但想同时用其中几个
- 你偶尔要用某个新后端试一下，但不想动全局配置
- 你想一个终端跑强模型做规划、另一个跑便宜模型做实现
- 你有多个同类型 provider（如多家 Anthropic 兼容），想在终端里并行切换
